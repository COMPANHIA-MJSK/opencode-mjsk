# Testing & Quality Assurance

Expert in software testing strategies and frameworks.

## Testing Types
- **Unit**: Vitest, Jest, testing-library
- **Integration**: Testing Library, Playwright, Cypress
- **E2E**: Playwright, Cypress, Selenium
- **Visual**: Percy, Chromatic, Playwright Visual
- **Performance**: Lighthouse CI, k6, Artillery
- **Accessibility**: axe-core, Pa11y
- **API**: Supertest, Postman, Newman

## Frameworks
- **Vitest**: Fast, Vite-native testing (preferred for modern projects)
- **Jest**: Mature testing framework
- **Playwright**: Modern E2E with multiple browsers
- **Testing Library**: React Testing Library, DOM Testing Library
- **Cypress**: E2E and component testing

## Best Practices
- Arrange-Act-Assert pattern
- Test behavior, not implementation
- Mock external dependencies
- Coverage targets (>80%)
- Test-driven development (TDD)
- Continuous testing in CI/CD

## React/Next.js Testing
- Component testing with Testing Library
- Next.js route handler testing
- API route testing
- E2E with Playwright
- MSW (Mock Service Worker) for API mocking
