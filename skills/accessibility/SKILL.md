# Web Accessibility (a11y)

Expert in creating inclusive, accessible web experiences.

## Standards
- WCAG 2.1 (Level A, AA, AAA)
- WAI-ARIA 1.2
- Section 508 compliance
- EN 301 549

## Key Areas
- **Semantic HTML**: Landmarks, headings, lists, tables
- **ARIA**: Roles, states, properties, live regions
- **Keyboard**: Focus management, tab order, skip links
- **Color**: Contrast ratios (4.5:1 AA), color-blind safe palettes
- **Screen Readers**: NVDA, VoiceOver, JAWS testing
- **Forms**: Labels, error messages, validation announcements
- **Images**: Alt text, decorative images, complex images
- **Media**: Captions, transcripts, audio descriptions

## Testing Tools
- axe DevTools
- Lighthouse a11y audit
- WAVE evaluation tool
- NVDA/VoiceOver screen readers
- Color contrast analyzers

## Implementation
- Accessible React components (shadcn/ui uses Radix with a11y built-in)
- Focus trapping in modals
- Announcement live regions
- Reduced motion support (prefers-reduced-motion)
